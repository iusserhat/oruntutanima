🔖 Classification (Sınıflandırma) Nedir?

Classification, ayrık (discrete/kategorik) bir çıktıyı tahmin etmek için kullanılan Supervised Learning yöntemidir. Sonuç belirli sınıflardan birine aittir.

Analoji: Bir doktor hastayı muayene eder ve "Kanser var mı yok mu?" diye karar verir. Bu bir classification problemi. Regresyondan farkı: "kanser ne kadar büyük?" diye sorsaydı o zaman regression olurdu.
👥

İkili Sınıflandırma (Binary)

Sadece 2 sınıf var: Evet/Hayır, 0/1, Olumlu/Olumsuz

  • Müşteri gidecek mi? (Churn: Evet/Hayır)
  • E-posta spam mi?
  • Kanser var mı?
🎯

Çok Sınıflı (Multi-class)

3 veya daha fazla sınıf: 0–9, renk kategorileri

  • El yazısı rakam tanıma (0,1,2...9)
  • Biyometrik tanıma
  • Hava durumu (güneşli/yağmurlu/karlı)

🔮 KNN (K-Nearest Neighbors) Algoritması

KNN, "Söyle bana arkadaşların kimler, sana kim olduğunu söyleyeyim" mantığıyla çalışır. Yeni bir veri noktası geldiğinde ona en yakın k komşusuna bakılır. O komşuların çoğunluğu hangi sınıfsa, yeni nokta da o sınıfa atanır.

📌
KNN'in Önemli Özelliği
"KNN'de model eğitimi YOKTUR! Tüm veri noktaları sistemde saklanır (instance-based learning). Yeni nokta gelince mevcut verilerle hesaplama yapılır." — Bu yüzden SVM ve Decision Tree gibi modellere kıyasla "lazy learner" denir.

KNN Adımları:

  1. Yeni veri noktası gelir
  2. Mevcut tüm eğitim noktalarına uzaklık hesaplanır
  3. En yakın k nokta seçilir
  4. Classification için: En çok tekrar eden sınıf seçilir (çoğunluk oyu)
  5. Regression için: k komşunun ortalaması alınır

🔢 K Değeri Nasıl Seçilir?

K değeri çok önemlidir. Ne çok küçük ne de çok büyük olmalı:

K = 1 → Kötü!

Sadece 1 komşuya bakılır. Overfitting riski yüksektir. Noise veriye çok duyarlıdır.

K çok büyük → Kötü!

Çok fazla komşuya bakılır. Underfitting riski. Model çok genelleşir, yerel örüntüleri kaçırır.

Doğru k: k=1'den 10'a kadar denenir (veya sqrt(n)). Her k için hata oranı (error rate) hesaplanır. En düşük hataya sahip k seçilir.

from sklearn.neighbors import KNeighborsClassifier
from sklearn import metrics

Ks = 10
mean_acc = np.zeros((Ks-1,))

for n in range(1, Ks):
    neigh = KNeighborsClassifier(n_neighbors=n).fit(X_train, y_train)
    yhat  = neigh.predict(X_test)
    mean_acc[n-1] = metrics.accuracy_score(y_test, yhat)

print("Best accuracy:", mean_acc.max(), "– K=", mean_acc.argmax()+1)

📏 Mesafe Metrikleri

KNN'de "en yakın" kavramı bir mesafe metriği gerektirir:

MetrikFormülNe Zaman?
Euclidean (Öklid) ⭐ En yaygın d = √[Σ(xᵢ − yᵢ)²] Sürekli veriler, default seçim
Manhattan (Taksi) d = Σ|xᵢ − yᵢ| Izgara benzeri alanlar, daha dayanıklı
Minkowski d = (Σ|xᵢ − yᵢ|^p)^(1/p) Genel form: p=2 → Euclidean, p=1 → Manhattan
Chebyshev d = max(|xᵢ − yᵢ|) Satranç hamleleri gibi uniform hareket
Euclidean (2D): d = √[(x₂ − x₁)² + (y₂ − y₁)²]

Euclidean (nD): d = √[Σᵢ (xᵢ − yᵢ)²]

🧮 Hesaplama Örneği — Adım Adım (Nottan)

Yeni nokta = (3, 3), k = 3. Hangi sınıfa giriyor?

NoktaÇalışma Saati (x₁)Deneme Puanı (x₂)SınıfEuclidean d = √[(3-x)²+(3-y)²]En Yakın 3?
P111B√[(3-1)²+(3-1)²] = √8 = 2.83
P221B√[(3-2)²+(3-1)²] = √5 = 2.24✅ 2.
P322B√[(3-2)²+(3-2)²] = √2 = 1.41✅ 1. (EN YAKIN)
P444A√[(3-4)²+(3-4)²] = √2 = 1.41✅ 1. (eşit)
P554A√[(3-5)²+(3-4)²] = √5 = 2.24
P645A√[(3-4)²+(3-5)²] = √5 = 2.24

En yakın 3 komşu: P3 (B), P4 (A), P2 (B)

Sınıf sayısı: B=2, A=1 → Çoğunluk B! → Yeni nokta B sınıfına girer.

🚨
Sınav: KNN Hesaplama Sorusu Formatı
"Mevcut noktalar ve sınıfları verilir → Yeni nokta için k=3 veya k=5 ile sınıfı bul" → Her noktanın uzaklığını hesapla, en yakın k tanesini bul, çoğunluk sınıfı seç.

🔄 KNN ile Regresyon

KNN sadece sınıflandırma için değil, sürekli değer tahmini (regresyon) için de kullanılabilir.

KNN — Classification

k komşunun çoğunluğuna ait sınıfı al → yeni noktanın sınıfını belirle.

Çıktı: "Churn" ya da "No Churn"

KNN — Regression

k komşunun ortalamasını al → yeni noktanın değerini tahmin et.

Çıktı: "Tahmin maaş: 5.400 TL"

📌
KNN'de LogLoss Hesabı (Nottan)
"KNN'de predict churn nasıl bulunur? (LogLoss için olasılık şekliyle)" — k=10 olsun. 10 komşunun değerlerine bakılır. (Churn=1 olanlar alınır). 9 tanesi 1, 1 tanesi 0 ise → 9/10 = 0.9 → Predicted churn = 0.9. 1 tanesi 1 ise → 1/10 = 0.1 → Predicted churn = 0.1. Ortalama LogLoss için bulunan değerlerin ortalaması alınır.

⚠️ KNN'de Normalizasyon Neden Zorunlu?

KNN mesafe hesapladığı için ölçek farklılıkları sonucu doğrudan etkiler:

Düşün: Feature 1 = yaş (0-100), Feature 2 = maaş (0-50.000). Normalizasyon olmadan maaş farkı yaş farkını tamamen ezer. Euclidean mesafe sadece maaş tarafından domine edilir ve yaş bilgisi anlamsız hale gelir.

Bu yüzden KNN, SVM, Logistic Regression gibi mesafe veya gradient bazlı algoritmalarda normalizasyon ZORUNLUDUR.

📉 LogLoss (Olasılıksal Değerlendirme)

LogLoss, modelin tahminlerinin 0-1 arasındaki olasılık şekliyle ifadesidir. Küçük olması daha iyidir.

LogLoss = −(1/n) · Σ[y · log(ŷ) + (1−y) · log(1−ŷ)]
📌
Nottan — LogLoss Örneği

Churn değeri 1 olan biri için Predicted Churn = 0.91 → LogLoss = 0.11 (iyi) ✅

Churn değeri 1 olan biri için Predicted Churn = 0.13 → LogLoss = 2.4 (kötü) ❌

Churn değeri 0 olan biri için Predicted Churn = 0.04 → LogLoss = 0.04 (iyi) ✅


"LogLoss ne zaman kullanılmalı? 0.49 ile 0.001 değerleri arasında fark olması iseniliyorsa LogLoss kullanılmalı (0-1 yerine olasılık kullanılacağı için)."

💻 KNN Kod Örnekleri

import numpy as np
import pandas as pd
from sklearn import preprocessing
from sklearn.model_selection import train_test_split
from sklearn.neighbors import KNeighborsClassifier
from sklearn import metrics

# ── 1. Veri Yükleme (Telecom Churn — 12 feature) ──
df = pd.read_csv('teleCust1000t.csv')
df.head()
df.columns       # Feature sayısını ver
df.shape         # Satır ve sütun sayısını ver
df['custcat'].value_counts()  # Sadece ayrık veriler için — bağımlı değişken dağılımı

# ── 2. Feature ve Label Ayır ──
X = df.drop('custcat', axis=1).values   # Tüm featurelar
y = df['custcat'].values               # Bağımlı değişken

# ── 3. Normalizasyon (KNN için ZORUNLU) ──
X = preprocessing.StandardScaler().fit(X).transform(X.astype(float))

# ── 4. Train/Test Split ──
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, random_state=4
)

# ── 5. En iyi K'yı bul ──
mean_acc = np.zeros((9,))
for n in range(1, 10):
    neigh = KNeighborsClassifier(n_neighbors=n).fit(X_train, y_train)
    yhat  = neigh.predict(X_test)
    mean_acc[n-1] = metrics.accuracy_score(y_test, yhat)

print("En iyi Accuracy:", mean_acc.max())
print("En iyi K:", mean_acc.argmax()+1)

# ── 6. KNN Model Eğitimi ve Tahmin ──
k = 7  # Bulunan en iyi k
neigh = KNeighborsClassifier(n_neighbors=k).fit(X_train, y_train)
yhat  = neigh.predict(X_test)

# ── 7. Değerlendirme ──
print("Train Accuracy:", metrics.accuracy_score(y_train, neigh.predict(X_train)))
print("Test Accuracy:",  metrics.accuracy_score(y_test, yhat))

# KNN Regression için:
from sklearn.neighbors import KNeighborsRegressor
knn_reg = KNeighborsRegressor(n_neighbors=3)
knn_reg.fit(X_train, y_train)

Bölüm 4 Sınav Özeti
  • Classification → ayrık çıktı, Regression → sürekli çıktı
  • KNN = en yakın k komşuya bak → çoğunluk sınıfı (Classification) veya ortalama (Regression)
  • K değeri for döngüsü ile denenerek seçilir
  • Euclidean mesafe: d = √[Σ(xᵢ-yᵢ)²]
  • KNN'de model eğitimi YOKTUR → Tüm veri hafızada tutulur
  • KNN için normalizasyon ZORUNLU (KNN, SVM, Logistic Regression)
  • LogLoss: 0-1 olasılık değeri, küçük olması iyidir
